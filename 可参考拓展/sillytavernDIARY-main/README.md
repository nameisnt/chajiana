## 🎯 简介

日记本插件是一个功能完善的 SillyTavern 扩展，为您的角色对话提供完整的日记管理系统。通过智能的AI交互和优雅的界面设计，让日记记录变得轻松而有趣。

## 💾 存储与迁移提示

日记本主要数据优先保存到 SillyTavern 的 `user/files` 独立 JSON 文件中：

- 普通日记：`user/files/diary-data.json`
- 交换日记：`user/files/diary-exchange-data.json`
- 回收站：`user/files/diary-recycle-bin.json`
- 自定义 API 设置：`user/files/diary-api-settings.json`
- 日记分组设置：`user/files/diary-groups.json`

如果文件读取或写入失败，插件会自动回退到原来的 `extension_settings.sillytavernDIARY` 数据，尽量避免数据丢失。

从旧版本升级时，建议先使用本体插件的“导出数据”做一次备份，再安装迁移插件把旧 settings 数据写入独立文件。迁移工具会合并写入，不会清空独立文件里已有的新日记。

迁移工具地址：<https://github.com/EtafCisky/diary-migration-tool>

### ✨ 核心特色

- 📝 **智能写日记** - AI后台生成格式化的日记内容，不影响聊天记录
- 🔌 **自定义 API** - 支持接入 OpenAI 兼容接口，自定义 URL、密钥与模型
- 🏷️ **日记备注** - 为普通日记添加目录专用副标题，方便快速区分内容
- 📁 **日记分组** - 为普通日记建立目录分组，支持加入、筛选和删除分组，分组列表保存在独立文件中
- 🧹 **普通日记批量删除** - 在角色日记目录中勾选多篇日记后统一删除
- 💌 **交换日记** - 与角色进行日记交换，记录彼此的生活点滴
- 🤖 **自动写日记** - 按楼层数自动触发，分角色独立计数，后台执行
- 🗑️ **智能回收站** - 失败内容自动保存，支持编辑和重新保存
- 📚 **日记本管理** - 按角色分类浏览和管理所有日记
- 🔄 **容错机制** - 解析失败或保存失败时自动保存到回收站
- 👤 **自定义角色** - 为任意角色写日记，不限于当前角色卡
- 🎲 **Reroll功能** - 为角色回复生成多个版本，选择最满意的
- 🎨 **主题切换** - 经典/简洁/夜间三种视觉风格随心切换
- 📱 **移动端优化** - 完整的响应式设计，支持各种屏幕尺寸
- 🎯 **悬浮窗设计** - 可拖拽的圆形悬浮窗，不影响聊天体验
- 📖 **在线文档** - 设置页面内置完整使用文档查看器

## 📦 安装说明

### 方法一：通过 SillyTavern 内置安装器（推荐）

1. 打开 SillyTavern，进入 **扩展** 页面
2. 点击 **安装插件** 按钮
3. 在弹出的输入框中粘贴插件仓库地址：

   ```
   https://github.com/EtafCisky/sillytavernDIARY
   ```

4. 点击确认，等待自动下载和安装
5. 重启 SillyTavern 或刷新页面
6. 在聊天页面应该能看到可拖拽的圆形悬浮窗（📖图标）

### 方法二：手动安装

1. 从 GitHub 下载插件：[https://github.com/EtafCisky/sillytavernDIARY](https://github.com/EtafCisky/sillytavernDIARY)
2. 将整个 `sillytavernDIARY` 文件夹复制到以下目录：

   ```
   SillyTavern/public/scripts/extensions/third-party/
   ```

3. 重启 SillyTavern 或在扩展管理页面点击 "重新加载"
4. 在聊天页面应该能看到可拖拽的圆形悬浮窗（📖图标）

### 方法三：Git 克隆

```bash
cd [SillyTavern安装目录]/public/scripts/extensions/third-party/
git clone https://github.com/EtafCisky/sillytavernDIARY.git sillytavernDIARY
```

然后重启 SillyTavern 或重新加载扩展。

### 验证安装

安装成功后，您应该能看到：

- ✅ 在"扩展"页面能看到"📖 日记本"插件
- ✅ 在聊天页面能看到圆形悬浮窗
- ✅ 点击悬浮窗能展开三个功能按钮（📖日记本、✏️写日记、🗑️回收站）

7. 在角色日记目录右上角点击 **🗑️** 可进入批量删除模式，勾选多篇日记后再次点击 **🗑️** 统一删除

## 📚 功能详解

### 1. ✏️ 写日记功能

#### 功能说明

引导AI按照标准格式创作日记，使用后台生成模式，不在聊天记录中留下痕迹，自动保存到独立 JSON 文件存储。

#### 使用步骤

1. **点击"写日记"按钮** → 弹出角色选择窗口
2. **选择角色**：
   - 输入自定义角色名：为任意角色写日记
   - 留空直接发送：使用当前角色卡名称
3. **后台生成** → 系统使用 `/gen` 命令在后台生成日记内容
4. **自动解析保存** → AI生成完成后自动提取日记内容并保存
5. **失败处理** → 如果解析或保存失败，内容自动保存到回收站

#### 日记格式

```
<日记>
标题：今天的心情
时间：2025年10月5日
内容：今天天气很好，和朋友一起去了公园...
</日记>
```

#### 注意事项

- ✅ 支持任意角色名，不限于当前角色卡
- ✅ 日记自动按角色分类存储
- ✅ 重复标题会覆盖旧日记
- ✅ 后台生成模式，不在聊天记录中留下痕迹
- ✅ 失败内容自动保存到回收站，可编辑后重新保存

### 2. 💌 交换日记功能

#### 功能说明

交换日记是一种更深入、更私密的情感分享方式。你可以与角色进行日记交换，记录彼此的生活点滴、心情感受和难忘时刻。这不是即时对话，而是一种异步的、更有仪式感的交流方式。

#### 核心特性

- **系列管理** - 为每个角色创建多个交换日记系列
- **条目追踪** - 每个系列包含多个日记条目，按时间顺序排列
- **双向记录** - 记录你的日记和角色的回复
- **Reroll功能** - 为角色回复生成多个版本（最多5个），选择最满意的
- **思维导图布局** - 条目以网格形式展示，带有连接线
- **移动端适配** - PC和移动端有不同的优化布局
- **删除管理** - 支持批量删除系列和条目

#### 使用流程

**1. 创建交换日记**

1. 点击悬浮窗下方的 **💌 交换日记** 按钮
2. 在封面页点击左上角"写一篇日记"或右下角箭头
3. 填写表单：
   - **角色名称**：留空使用当前角色，或输入自定义角色名
   - **选择系列**：选择现有系列或创建新系列
   - **系列名称**：创建新系列时可自定义名称（留空自动生成）
   - **日记时间**：可选，如"午后，阳光正好"
   - **日记内容**：写下你想分享的内容
4. 点击"提交"，日记保存成功

**2. 查看交换日记**

1. 在封面页点击右下角箭头进入角色列表
2. 选择角色查看该角色的所有系列
3. 点击系列卡片进入条目列表
4. 条目以网格形式展示，显示：
   - 条目编号和日期
   - 你的日记预览
   - 角色回复预览（如有）
5. 点击条目卡片进入阅读视图

**3. 阅读视图**

- **PC端**：左右分栏显示
  - 左侧：你的日记
  - 右侧：角色的回复
- **移动端**：单页显示
  - 奇数页：你的日记
  - 偶数页：角色的回复
- **翻页**：使用左下角/右下角的箭头按钮
- **页码跳转**：底部中央的页码输入框

**4. Reroll功能**

当角色回复后，标题旁会出现 **⟳** 按钮：

1. 点击 **⟳** 按钮打开版本选择器
2. 查看所有已生成的版本（显示完整内容）
3. 点击 **⟳ 生成新版本** 按钮生成更多版本（最多5个）
4. 选择满意的版本（点击卡片）
5. 点击"确认选择"保存

**5. 删除功能**

- **删除系列**：
  1. 在系列列表页点击右上角垃圾桶图标
  2. 勾选要删除的系列
  3. 再次点击垃圾桶图标确认删除
  4. 点击"×取消"按钮退出删除模式

- **删除条目**：
  1. 在条目列表页点击右上角垃圾桶图标
  2. 勾选要删除的条目
  3. 再次点击垃圾桶图标确认删除
  4. 点击"×取消"按钮退出删除模式

#### 交换日记格式

角色回复使用以下格式：

```
<交换日记>
标题：[简短的标题，概括主要事件或心情]
时间：[写日记的时间]
内容：[日记正文，记录最近发生的事情和感受]
</交换日记>
```

#### 触发机制

角色回复会在以下情况自动触发：

- 你写完日记后，系统会在配置的楼层范围内随机触发
- 默认触发范围：1-10楼层
- 可在设置中调整触发范围

#### 移动端优化

- **条目列表**：标题居中在上方，条目网格在下方
- **阅读视图**：单页显示，左右滑动翻页
- **操作按钮**：优化位置，避免与关闭按钮重叠
- **Reroll弹窗**：全屏显示，更好的阅读体验

#### 注意事项

- ✅ 每个系列可以包含无限个条目
- ✅ 每个条目可以有最多5个角色回复版本
- ✅ 删除系列会删除其中的所有条目
- ✅ 删除条目不会影响其他条目
- ✅ Reroll生成的版本会保存，可随时切换
- ✅ 未选中的版本不会被删除，可以随时查看

### 3. 🗑️ 回收站功能

#### 功能说明

智能回收站系统，自动保存失败的AI生成内容，提供编辑和重新保存功能。适用于：

- 写日记时AI生成的内容格式不正确
- 日记保存到文件存储失败
- 需要手动编辑和调整日记内容
- 批量处理和管理失败内容

#### 使用步骤

1. **自动保存** → 系统在以下情况自动保存内容到回收站：
   - 日记格式解析失败
   - 文件存储保存失败
   - 系统生成错误
2. **查看回收站** → 点击 **🗑️ 回收站** 按钮
3. **按角色分组** → 回收站按角色分组显示，默认收起
4. **编辑内容** → 点击条目查看详情，可编辑内容
5. **重新保存** → 编辑后点击保存按钮重新保存为日记
6. **删除条目** → 不需要的内容可直接删除

#### 回收站管理

- **角色分组** - 所有条目按角色名分组显示
- **可展开收起** - 点击角色组标题展开/收起该组条目
- **内容预览** - 显示失败原因和内容摘要
- **批量清空** - 一键清空所有回收站内容
- **编辑保存** - 修复格式问题后重新保存为正式日记

### 4. 📖 日记本功能

#### 浏览日记

1. **打开日记本** → 点击"日记本"按钮
2. **封面页** → 显示统计信息：
   - 📊 日记总数
   - 👥 角色数量
3. **角色列表** → 点击"打开日记本"进入
   - 显示所有有日记的角色
   - 每个角色卡片显示日记数量
4. **日记列表** → 点击角色卡片查看该角色的日记
   - 按时间排序显示
   - 分页浏览（每页10篇）
5. **日记详情** → 点击日记卡片查看完整内容
   - 显示标题、时间、内容
   - 可通过左上角 **⚙️ 功能按钮** 设置备注副标题
   - 支持上一页 / 下一页顺序翻阅
   - 支持删除操作

#### 删除日记

单篇删除：

1. 在日记详情页
2. 点击 **🗑️ 删除日记** 按钮
3. 确认删除
4. 日记从当前存储中永久删除

批量删除：

1. 在角色的日记目录页，点击右上角 **🗑️** 图标进入批量删除模式
2. 勾选需要删除的日记卡片
3. 再次点击右上角 **🗑️** 图标并确认删除
4. 如需退出批量删除模式，点击右上角 **×** 按钮

#### 分组功能

- **创建分组** - 在角色日记目录右上角点击 **📁**，选择“创建分组”并输入名称
- **加入分组** - 选择“勾选日记加入分组”，选定目标分组后勾选日记，再点击右上角确认按钮
- **移动规则** - 每篇日记同一时间只能属于一个分组，加入新分组会从旧分组移动过去
- **筛选分组** - 选择“筛选”后从可搜索列表中选择分组，只显示该分组中的日记；选择“全部日记”可取消筛选
- **删除分组** - 选择“删除分组”只会移除分组和日记的分组归属，不会删除其中的日记

#### 导航操作

- **返回按钮** - 返回上一级页面
- **分页按钮** - 浏览更多日记
- **⚙️ 功能按钮** - 固定在左上角，展开备注等附加操作
- **详情翻页按钮** - 固定在日记详情页左右两端，用 `<>` 圆形按钮快速翻页
- **关闭按钮** - 关闭日记本弹窗

#### 备注功能

- **备注副标题** - 可为每篇普通日记单独设置一个备注副标题
- **仅目录显示** - 备注只显示在日记目录页，不会插入日记正文内容
- **快捷管理** - 在详情页点击左上角 **⚙️ 功能按钮**，可设置、修改或清除备注

### 7. 🤖 自动写日记功能

#### 功能说明

自动写日记功能可以在聊天达到指定楼层数时，自动触发日记生成。无需手动操作，让日记记录更加自然流畅。

#### 配置步骤

1. 打开 **扩展设置** → **📖 日记本**
2. 在 **自动写日记** 区域找到 **触发间隔（楼层数）** 输入框
3. 输入数字N（建议使用偶数，如10、12、16）
4. 系统立即启用，显示状态信息

#### 工作原理

- **楼层计数**：每条消息（包括用户和AI）都计入楼层数
- **触发时机**：当前楼层 - 上次触发楼层 ≥ N时自动触发
- **起始楼层**：输入N时的当前楼层作为起始点
- **分角色独立**：每个角色单独计数和记录
- **智能检测**：等待AI回复完成后才触发，避免冲突

#### 使用示例

假设您在第15楼设置间隔为10：

- 起始楼层：15
- 第一次触发：第25楼（15+10）
- 第二次触发：第35楼（25+10）
- 以此类推...

#### 状态显示

- **功能未启用** - 未设置间隔或间隔为0
- **已启用，还需X条消息触发** - 显示距离下次触发的剩余楼层数
- **已达触发条件** - 已达到触发条件，等待合适时机执行

#### 禁用方法

在触发间隔输入框中删除数字或输入0即可禁用。

#### 注意事项

- ✅ 建议输入偶数，避免在用户发送消息后立即执行
- ✅ 功能启用后会在后台每3秒检查一次
- ✅ 检测到AI正在生成时会自动跳过，等待下次检查
- ✅ 切换角色时，每个角色独立计数
- ✅ 使用后台生成模式，不在聊天记录中留下痕迹
- ✅ 失败内容自动保存到回收站，可编辑后重新保存

## 🐛 故障排查

### 主题相关问题

#### 主题切换后样式不生效

1. 检查浏览器开发者工具的 Network 标签，确认 CSS 文件已加载
2. 检查 CSS 文件路径是否正确
3. 清除浏览器缓存后重试
4. 查看控制台是否有 CSS 语法错误

#### 主题选择不保存

1. 检查浏览器的 LocalStorage 是否被禁用
2. 查看控制台是否有保存错误信息
3. 尝试重启 SillyTavern

### 功能相关问题

#### 写日记后没有保存

1. 检查 AI 回复是否包含正确的日记格式
2. 打开控制台查看详细日志
3. 检查 `user/files/diary-data.json` 是否存在，或查看 `window.diaryFileStorageStatus()`
4. 尝试使用"记录"功能手动保存

#### 日记本打开后无内容

1. 确认已经写过至少一篇日记
2. 检查 `user/files/diary-data.json` 中是否有数据
3. 查看控制台是否有加载错误

#### 悬浮窗不显示

1. 检查扩展是否正确安装
2. 在设置中点击 "显示/隐藏悬浮窗"
3. 点击 "重置悬浮窗位置"
4. 刷新页面

#### AI 生成的日记格式不正确

1. 检查是否配置了专用的日记预设
2. 在预设中添加日记格式说明
3. 使用“记录”功能可以解析任意包含标准格式的内容

### 获取帮助

如果问题仍未解决：

1. 打开浏览器开发者工具（F12）
2. 查看 Console 标签中的错误信息
3. 复制错误信息并报告问题
4. 提供操作步骤和截图
