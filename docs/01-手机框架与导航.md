# 01 — 手机框架与导航：实现细节

> 对应 `酒馆手机功能与实现方案.md` §2（手机框架）
> 本文描述手机壳 UI、顶栏、主屏、悬浮球、导航栈、主题和生命周期的工程实现约束。

---

## 1. 手机壳外观

### 1.1 尺寸

| 环境 | 宽度 | 高度 |
|------|------|------|
| 桌面端 | `360px` | `700px` |
| 移动端 | `92vw`（最大 360px） | `85vh`（最大 700px） |

### 1.2 样式约束

- 圆角 `44px`（移动端 `28px`）
- `overflow: hidden` 确保内容不溢出圆角
- 无遮罩层 — 旧方案的半透明背景遮罩已废弃
- 无状态栏 — 不渲染时间、信号、Wi-Fi、电量
- 无底部 Home 条

### 1.3 弹窗挂载

手机根节点直接追加到 `document.body`：

```html
<div id="phone-creative-root">
  <div class="phone-root" data-theme="light|dark">
    <section class="phone-shell">...</section>
  </div>
</div>
```

- 根节点固定 ID `phone-creative-root`
- 使用 `aria-label="酒馆手机创作助手"`
- 所有 DOM ID/class 使用 `phone-creative-` 或 `pc-` 前缀
- `data-theme` 属性控制亮色/暗色主题

### 1.4 首次创建与后续显隐

- 首次打开时 `createApp(App).use(createPinia()).mount(root)` 创建 Vue 实例
- 之后只通过 `root.removeAttribute('hidden')` / `root.setAttribute('hidden', '')` 控制显隐
- 不重复挂载 Vue 应用

### 1.5 视觉设计

现代简约风格，大面积留白，克制使用色彩和阴影，强调内容本身。

**色彩**：

| 模式 | 底色 | 卡片 | 文字 |
|------|------|------|------|
| 亮色 | `#f2f2f7` 浅灰 | `rgba(255,255,255,0.8)` 毛玻璃 | `#1c1c1e` |
| 暗色 | `#1c1c1e` 深灰 | `rgba(255,255,255,0.08)` 半透明 | `#f5f5f7` |

- 强调色：`#007aff`（iOS 蓝），仅用于可交互元素
- 毛玻璃卡片：`backdrop-filter: blur(20px)` + 半透明背景 + 微边框 `rgba(255,255,255,0.1)`

**卡片与间距**：

- 卡片圆角 `16px`，按钮胶囊形（`border-radius: 999px`）
- 间距基准 `4px`，内边距三档：`12px`（紧凑）/ `16px`（常规）/ `20px`（宽松）
- 内容区留足呼吸空间，卡片间距 `12px`

**字体**：

字体通过 CSS 变量统一控制，用户可在设置中导入替换：

```css
:root {
  --phone-font-sans: -apple-system, BlinkMacSystemFont,
    "Segoe UI", Roboto, "Helvetica Neue", Arial,
    "Noto Sans SC", sans-serif;
  --phone-font-mono: "SF Mono", "JetBrains Mono",
    "Fira Code", "Cascadia Code", monospace;
}

.phone-root {
  font-family: var(--phone-font-sans);
}
```

- 设置中提供「字体」配置项，用户输入字体名称后即时覆盖 `--phone-font-sans` 变量
- 支持系统已安装字体名（如 `"LXGW WenKai"`、`"Noto Serif SC"`）
- 支持 Web 字体：用户填入 `@import` URL 或 CSS `@font-face` 片段，插件动态注入 `<style>` 节点后应用
- 提供「恢复默认」按钮

**阴影**：

- 卡片悬浮：`0 2px 12px rgba(0,0,0,0.08)`
- 手机壳：`0 8px 32px rgba(0,0,0,0.16)`
- 暗色模式下阴影改为纯黑 + 降低透明度

**图标**：

- 使用 Font Awesome 6 free solid（酒馆已加载）
- App 图标统一 `fa-solid` 风格，尺寸 `24px`

**过渡动画**：

- 页面切换：`200ms ease-out`，微位移动效（上移 `8px` + 淡入）
- 弹窗/抽屉：`250ms cubic-bezier(0.2, 0.9, 0.2, 1)`
- 日夜切换：`300ms` 背景色 + 文字色过渡

## 2. 顶栏

### 2.1 布局

```
┌──────────────────────────────────┐
│ ← 返回   当前功能名   🏠 🌙     │
└──────────────────────────────────┘
```

### 2.2 按钮行为

| 位置 | 元素 | 栈深 > 1 | 栈深 = 1 |
|------|------|----------|----------|
| 左侧 | 返回按钮 | `nav.back()` 返回上一页 | 关闭手机 `hidePhone()` |
| 中间 | 功能名 | 纯文本显示，不可交互 | — |
| 右侧 | 主页按钮 | `nav.home()` 回到主屏 | — |
| 右侧 | 日夜切换 | 切换 `data-theme` | — |

### 2.3 实现注意

- 顶栏高度固定，不随内容滚动
- 返回时保留原页面的列表筛选状态、滚动位置
- 从收藏进入详情后，返回应回到收藏列表（通过 `navState.origin` 判断）

## 3. 主屏

### 3.1 布局

- 分页网格展示 App 图标（桌面端 4 列，移动端 3 列）
- 底部 Dock 区最多容纳 4 个 App 图标
- 图标来源：Font Awesome 或内置 SVG

### 3.2 App 布局持久化

```ts
interface HomeScreenLayout {
  appOrder: string[];        // App ID 排序
  hiddenApps: string[];      // 隐藏的 App ID
  dockApps: string[];        // Dock 中的 App ID
}
```

- 布局保存在 `preferences` 全局逻辑文件
- 新版本新增 App 按 `defaultOrder` 插入，未知旧 ID 保留不显示
- "重置布局"恢复为所有 App 的 `defaultOrder/defaultDock/defaultVisible`

## 4. 悬浮球

### 4.1 DOM 结构

```html
<div class="pc-float-ball-root">
  <button type="button" class="pc-float-ball"
    title="打开酒馆手机" aria-label="打开酒馆手机">
    <i class="fa-solid fa-mobile-screen-button"></i>
  </button>
</div>
```

- 悬浮球独立挂载在 SillyTavern 主页面，不放在 Vue 根节点内
- 使用真实 `<button>` 元素，非 `<div>`

### 4.2 层级

```
手机弹窗 z-index: 2147483647
悬浮球   z-index: 2147482000  ← 低于弹窗
```

- 根容器 `pointer-events: none`，仅按钮接收事件
- 手机打开时主动隐藏悬浮球（`hidden` 属性或 `display: none`）

### 4.3 拖拽（Pointer Events 统一处理）

```
pointerdown → 记录起点 + setPointerCapture()
  → pointermove（超过阈值才进入拖拽状态）
    → 桌面鼠标/触控笔 > 5px，触摸 > 15px
    → 更新 left/top，clamp 在可视区域内
  → pointerup/pointercancel → 释放捕获 + 保存位置
    → 仅非拖拽态才触发 click → 打开手机
```

- 默认位置：右下区域，避开底部 ~100px
- 默认尺寸 `44px`（范围 `28–80px`）
- 默认颜色 `#007aff`
- 触摸结束后保留 ~300ms 防误触窗口
- resize/orientationchange 后重新 clamp

### 4.4 设置同步

```ts
interface FloatBallSettings {
  floatBallEnabled: boolean;
  floatBallSize: number;
  floatBallColor: string;
  floatBallX?: number;
  floatBallY?: number;
}
```

- 启用/关闭即时创建/删除
- 尺寸/颜色变化只更新 CSS 变量，不重建按钮
- `focus`、`pageshow`、`resize`、`orientationchange`、`visibilitychange` 时校正位置

## 5. 导航

### 5.1 栈结构

```ts
interface NavState {
  appId: string;
  page: string;
  title: string;
  params?: Record<string, string>;
  origin?: 'home' | 'favorites';
}
```

- `openApp(appId)` — 推入 App 首页
- `push(page, params)` — 推入子页
- `replace(page, params)` — 替换当前页
- `back()` — 弹出当前页；栈深 1 时关闭手机
- `home()` — 清空栈回到主屏

### 5.2 参数传递

- 导航参数只传稳定 ID（entityId、bookId 等），不传完整正文
- `origin: 'favorites'` 标记从收藏进入，返回时恢复收藏列表状态

### 5.3 状态保留

- `back()` 不丢失列表筛选、滚动位置
- 组件使用 `<KeepAlive>` 缓存列表页滚动状态

## 6. 主题

### 6.1 亮色/暗色

- 通过 `<div data-theme="light|dark">` 切换
- 主题样式使用 CSS 变量 + `[data-theme]` 选择器
- 主题偏好持久化到 `preferences` 全局逻辑文件
- **Tailwind 注意**：模板 `tailwind.config.js` 使用 `darkMode: 'class'`，手机插件改用 `data-theme` 属性控制主题。实现时须将 Tailwind 配置改为 `darkMode: ['selector', '[data-theme="dark"]']`，使 `dark:` 前缀工具类能响应 `data-theme` 而非 `.dark` class

### 6.2 壁纸

- 支持预设壁纸和自定义上传
- 自定义壁纸限制：`≤10 MiB`、`≤8192×8192`、仅 PNG/JPEG/WebP/GIF
- 壁纸原文件存储为 `user/files/phone-wallpaper-{id}.{ext}`
- Object URL 在更换、删除、组件卸载时 `revokeObjectURL()`
- 正式显示使用已上传的 `user/files` URL，不把 base64 常驻 Store

## 7. 生命周期

### 7.1 状态机

```
插件加载 → APP_READY → 等待 TavernHelper
  → 注册扩展菜单入口
  → 根据设置创建/不创建悬浮球

点击入口/悬浮球 → showPhone()
  → 首次：创建 Vue 根节点 + 挂载
  → 后续：显示根节点、隐藏悬浮球

关闭 → hidePhone()
  → 卸载 iframe、保留导航栈和表单
  → 根据设置恢复悬浮球
  → 刷新变更领域的 settings 兜底

销毁 → destroyPhone()
  → 停止活跃任务、恢复楼层
  → 卸载 Vue、移除节点和样式
  → 清理监听器、计时器
```

### 7.2 关键约束

- `showPhone()` 与 `hidePhone()` 分离；关闭 ≠ 销毁
- 关闭仅隐藏 + 卸载 iframe，保留 Vue 实例、导航栈、表单状态
- 销毁用于插件禁用、热重载或页面卸载
- `ensurePhone()` 幂等：已存在则复用，不存在则创建
- 初始化异常统一捕获，不在 `document.body` 不可用时留下半成品节点

### 7.3 监听管理

- 所有 `eventSource.on`、DOM 监听、窗口事件必须保存解除函数
- 解除函数统一放入 `disposers` 数组，销毁时全部执行
- 重复加载不重复注册监听

### 7.4 热重载兼容

- 全局防重键 `window.__sillytavernPhonePluginState__` 记录初始化状态
- 重载时等待 `destroyPhone()` 完成后再重新初始化
- 初始化失败复位状态，允许重试
