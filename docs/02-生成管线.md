# 02 — 生成管线：实现细节

> 对应 `酒馆手机功能与实现方案.md` §6（通用生成能力）
> 本文描述生成服务、来源楼层、隐藏事务、SendGuard、拼接模板、XML 解析、流式传输和文本通道的工程实现约束。

---

## 1. 生成服务

### 1.1 唯一入口

全插件只有一个 `GenerationService`，暴露统一接口 `generateContent(adapter, config)`。任何 App 不得：

- 直接调用 `window.TavernHelper.generate()`
- 自行实现流式监听、停止、楼层隐藏/恢复
- 维护第二套生成状态机

### 1.2 Adapter 契约

App 通过注册 adapter 提供业务差异：

```ts
interface GenerationAdapter<TConfig, TResult> {
  appId: string;
  actionId: string;           // 如 'normal' | 'readReaction' | 'newThread' | 'continueReply'
  configSchema: z.ZodType<TConfig>;
  buildRequest(config: TConfig): GenerationRequestParts;
  parse(raw: string, config: TConfig): ParseResult<TResult>;
  save(result: TResult, context: GenerationSaveContext): Promise<{ entityId: string }>;
}
```

关联类型：

```ts
// buildRequest 返回的声明式区段
interface GenerationRequestParts {
  extraContext?: string;      // 额外上下文（续写前文、回信历史等）
  references?: string;        // 引用素材
  appPrompt?: string;         // App 级提示词
  typePrompt?: string;        // 类型提示词（番外/小剧场）
  userRequirement?: string;   // 用户额外要求
  outputFormat: string;       // 固定 XML 输出格式（解析器维护，不可用户编辑）
}

// parse 返回的解析结果
type ParseResult<T> =
  | { ok: true; data: T; raw: string; warnings: string[] }
  | { ok: false; raw: string; warnings: string[] };

// save 的参数上下文
interface GenerationSaveContext {
  scopeId: string;
  source: SourceSelection;
  rawOutput: string;
}
```

- `appId + actionId` 全局唯一，注册时校验
- `buildRequest()` 只返回声明式区段，不直接调用 Tavern Helper、改楼层或订阅全局事件

### 1.3 生成流程

```
1. 校验 adapter + config（zod）
2. Token 预检（来源楼层 + 引用 + user_input vs 可用窗口）
3. 获取来源隐藏互斥锁，若已锁定则拒绝
4. SendGuard 激活 → 拦截酒馆 UI 发送
5. 来源隐藏事务：
   a. 快照全部楼层 message_id + is_hidden
   b. 持久化恢复日志
   c. 临时隐藏来源范围外的可见楼层
   d. 调用 generate()
   e. finally：按快照逐层恢复隐藏状态
   f. 核验身份哈希 → saveChat()
   g. 清除恢复日志
6. SendGuard 解除
7. 解析返回 XML
8. resultMode === 'preview' → 预览页
   resultMode === 'save'   → 直接保存并跳转详情
```

### 1.4 并发控制

- 全插件一把来源隐藏互斥锁
- 启动前若本插件或外部已有生成任务 → 拒绝
- 生成互斥锁释放必须在楼层恢复完成之后

## 2. 来源楼层选择

### 2.1 六种模式

| 模式 | 说明 |
|------|------|
| `latest` | 最新楼层 |
| `fromStart` | 从 0 至指定楼层 |
| `all` | 全部楼层 |
| `single` | 指定单层 |
| `recent` | 最近 N 楼 |
| `range` | 自定义范围 |

```ts
interface SourceSelection {
  scopeId: string;
  chatIdAtGeneration: string;
  mode: 'latest' | 'fromStart' | 'all' | 'single' | 'recent' | 'range';
  ranges: Array<{ start: number; end: number }>;
  messageIds: number[];
  label: string;
  sortKey: number;
}
```

### 2.2 读取规则

- 只读可见楼层、只取当前选中 swipe
- 不发送隐藏楼层或未选中 swipe
- 不重新组装 `{role, content}`，不改写正文
- 不附加楼层号或发言者标签
- 范围解析后无任何可见楼层 → 阻止生成并提示

## 3. 来源楼层隐藏事务

### 3.1 事务步骤

```
1. 快照全部楼层 message_id + is_hidden 状态
2. 将快照（含 contentHash）持久化为恢复日志
3. 只把「原本可见且不在来源集合内」的楼层临时设为隐藏
4. 原本隐藏的楼层始终保持隐藏
5. 来源集合已覆盖全部可见楼层 → 跳过隐藏
6. 调用 generate()
7. finally：按快照逐层恢复原始隐藏状态
8. 核验身份哈希与快照匹配
9. saveChat() 落盘
10. 清除恢复日志
```

### 3.2 关键约束

- 隐藏/恢复统一使用 `setChatMessages(..., { refresh: 'none' })`
- 临时隐藏阶段**禁止**调用 `saveChat()`
- 恢复必须写在 `finally` 中 — 成功、停止、失败、解析异常都执行
- 恢复时按快照逐层还原，不能简单把所有楼层取消隐藏
- 事务绑定 `scopeId` + 任务 ID；恢复前核对当前聊天 scopeId

### 3.3 崩溃恢复日志

```ts
interface PendingVisibilityRecovery {
  scopeId: string;
  generationId: string;
  createdAt: string;
  messages: Array<{
    messageId: number;
    isHidden: boolean;
    role: 'system' | 'assistant' | 'user';
    name: string;
    contentHash: string;  // SHA-256 of role + name + 当前swipe正文
  }>;
}
```

- 保存在 `extension_settings.sillytavernPhone.pendingVisibilityRecoveries`
- 必须先持久化成功，才能隐藏楼层
- `contentHash` 使用 Web Crypto SHA-256
- 恢复日志超 settings 保存上限 → 阻止本次生成
- 身份哈希不匹配 → 保留日志进入人工恢复，不盲目写回
- 插件启动和聊天切换后自动检查并核对恢复

### 3.4 跨聊天安全

- 生成期间发生聊天切换 → 立即停止生成
- 不在新聊天执行恢复
- 保留恢复日志，原聊天再次激活时优先恢复
- 恢复前核对 `chat_metadata.scopeId`，禁止跨聊天恢复

## 4. SendGuard

### 4.1 职责

生成事务期间拦截用户通过酒馆 UI 发起的发送操作，但不影响其他扩展或酒馆自身按钮状态。

### 4.2 实现约束

- 添加带插件命名空间的**捕获阶段**事件监听
- 拦截酒馆发送按钮点击和聊天输入框 Enter
- 显示提示遮罩
- **不调用**全局 `deactivateSendButtons()`/`activateSendButtons()`
- 事务结束只移除自己的监听和遮罩，不恢复其他扩展的按钮状态
- 检测到非本任务的生成（其他扩展触发）→ 立即停止手机任务并先恢复楼层

## 5. user_input 拼接

### 5.1 拼接模板

来源楼层由酒馆原生聊天历史机制处理（不拼入 `user_input`）。`user_input` 按以下顺序拼接：

```
额外上下文（如有）
  → 引用素材（如有）
  → App 提示词
  → 类型提示词（如有）
  → 用户额外要求
  → 固定输出格式（XML 模板，由解析器维护，不可用户编辑）
```

空的区段直接省略。

### 5.2 各动作的额外上下文

| App | 动作 | 额外上下文 |
|-----|------|-----------|
| 日记 | 让他人阅读 | 被阅读的日记正文 |
| 论坛 | 续回 | 主楼 + 现有回复（含楼中楼关系） |
| 番外 | 续写 | 续写点前全部章节（书级总结可替代已覆盖部分）|
| 番外 | 重写 | 原章节正文 |
| 书信 | 回信 | 最近 N 封相关往返信件 |
| 其他 | 普通生成 | 无 |

### 5.3 引用素材

- 按内容类型和分册分组选择
- 支持搜索、展开/收起、分组全选
- 引用放入 `user_input` 参考资料区，不伪装成聊天楼层

## 6. 生成调用

### 6.1 generate() 约束

```ts
generate({
  user_input: assembledInput,
  should_silence: true,       // 不污染酒馆正常回复
  should_stream: streamMode,
  generation_id: uniqueId,    // 用于流式过滤和停止
  // 不传 preset_name、injects、ordered_prompts、overrides.chat_history
  // 默认通道不传 custom_api
  // 外部通道仅附加 custom_api
})
```

- 不覆盖 `chat_history` — 让酒馆从临时可见的真实楼层构建聊天历史
- 不传预设、世界书文本或额外注入
- `should_silence: true` — 手机生成不占用酒馆正常回复位置

### 6.2 Token 预检

生成前对「实际来源楼层 + 引用内容 + 完整 user_input」估算 token，为当前预设/世界书/输出至少保留 20% 上下文余量。超过可用窗口时阻止生成并列出各区段占用。

## 7. 流式传输

### 7.1 模式组合

`resultMode` 和 `stream` 是两个独立配置：

| resultMode | stream | 行为 |
|------------|--------|------|
| `preview` | `true` | 显示实时文本 → 结束后进预览 → 确认保存 |
| `preview` | `false` | 等待完成 → 解析 → 进预览 → 确认保存 |
| `save` | `true` | 显示实时文本 → 结束后解析 → 直接保存 |
| `save` | `false` | 等待完成 → 解析 → 直接保存 |

### 7.2 停止处理

- 用户停止后**不**按 `resultMode: 'save'` 自动保存
- 已有流式文本 → 作为未完成草稿进入解析修复
- 无可用文本 → 仅记录停止状态

### 7.3 流式事件过滤

- 按 `generation_id` 过滤流式事件
- 停止调用 `stopGenerationById(generation_id)`
- 并发任务间不串流式内容

## 8. XML 解析

### 8.1 统一输出格式

所有创作 App 共用最外层 `<result>` 标签：

```xml
<!-- 日记 / 番外章节 / 小剧场 / 总结 -->
<result>
  <title>标题</title>
  <content>正文（小剧场 frontend 模式时为 HTML）</content>
</result>

<!-- 论坛首帖 -->
<result>
  <board>板块名</board>
  <title>帖子标题</title>
  <author>作者</author>
  <content>主楼正文</content>
  <replies>
    <reply><author>作者</author><content>回复</content><parent_ref></parent_ref></reply>
  </replies>
</result>

<!-- 书信 -->
<result>
  <title>标题</title>
  <content>正文</content>
</result>
```

### 8.2 解析规则

- 提取第一个完整 `<result>`，忽略外围说明文字
- 多个 `<result>` 或缺少必需字段 → 进入解析修复
- 小剧场 frontend 模式：以 `<content>` 起止标签边界提取原始 HTML
- 论坛临时引用（`E1/E2/N1/N2`）在保存时转为真实 `parentReplyId`
- 自引用/循环/不存在引用 → 降级为普通回复并警告

### 8.3 解析失败处理

- 不丢原文，作为 `FailedGenerationDraft` 保存
- 预览页展示原始输出 + 解析警告
- 用户可编辑原文后重新解析

```ts
interface FailedGenerationDraft {
  id: string;
  contentType: ContentType;
  rawOutput: string;
  warnings: string[];
  source: SourceSelection;
  createdAt: string;
}
```

## 9. 文本生成通道

### 9.1 两种通道

```ts
type TextProvider =
  | { mode: 'tavern' }
  | { mode: 'external'; apiUrl: string; apiKey?: string; model: string;
      contextWindow?: number; maxOutputTokens?: number };
```

- **tavern**：`generate()` 不传 `custom_api`，使用酒馆当前 API/模型/采样参数
- **external**：`generate({ custom_api: { apiurl, key, model, source: 'openai', max_tokens? } })`

### 9.2 URL 规范化

`normalizeExternalApiUrl()`：去首尾空白、末尾 `/`、末尾 `/models`、末尾 `/chat/completions`。

### 9.3 安全约束

- URL 只允许 `http:`/`https:` 协议
- 拒绝内嵌用户名密码、`file:`、`javascript:`、`data:` 等
- API Key 只能进入独立字段，不拼进 URL 或日志
- 界面默认遮罩，日志不写 Key
- 连接测试通过 `TavernHelper.getModelList({ apiurl, key })` 代理
- 默认导出 Key 替换为空字符串
- 外部 API 未配置或测试失败时阻止生成
