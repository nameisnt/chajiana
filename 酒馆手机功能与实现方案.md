## 1. 产品定位

酒馆手机是一个运行在 SillyTavern 中的**手机式创作工具箱**，以第三方扩展插件形式安装。插件提供手机壳式的 UI 容器，内部包含多个创作与管理 App。所有 AI 生成复用酒馆当前的角色卡、预设、世界书和文本 API。

插件只负责：组织聊天素材 → 发起文本生成 → 解析结果 → 保存创作内容。

**不负责**：角色卡管理、预设管理、世界书管理、生图、相册、角色画像、世界观管理。

## 2. 手机框架

### 2.1 外观

- 手机壳式 UI 容器，约 `360×700`（桌面端）/ `92vw×85vh`（移动端）
- 亮色/暗色主题切换
- 圆角外壳
- **无遮罩层** — 手机弹窗直接显示在页面上，不设半透明背景遮罩
- **无状态栏** — 不需要时间、信号、Wi-Fi、电量等装饰元素
- **无底部 Home 条** — 不需要底部横条装饰

### 2.2 顶栏

顶栏是手机的核心功能区，包含以下元素：

| 位置 | 功能 | 说明 |
|------|------|------|
| 左侧 | 返回/关闭 | 栈深度 >1 时返回上一页；栈深度 =1 时关闭手机 |
| 中间 | 当前功能名 | 显示当前所在 App 或页面名称 |
| 右侧 | 主页按钮 | 一键回到手机主屏 |
| 右侧 | 日夜切换 | 切换亮色/暗色主题 |

### 2.3 主屏

- 分页网格展示 App 图标
- 底部 Dock 区（可放置常用 App）
- App 可排序、显隐、加入/移出 Dock
- 布局持久化

### 2.4 入口

- 酒馆扩展菜单注册「打开酒馆手机」
- 可选悬浮球（可在设置中开启/关闭、设置尺寸和颜色）
- 悬浮球：可拖拽（Pointer Events 统一处理鼠标/触摸/触控笔），位置持久化
- 手机已打开时悬浮球自动隐藏

### 2.5 导航

栈式导航：

```text
主屏 → 打开 App → 推入子页 → 替换页 → 返回上一页 → 回主屏
```

- 返回时不丢失列表筛选、滚动位置
- 从收藏进入详情后，返回应回到原收藏列表状态
- 导航参数只传稳定 ID，不传完整正文

## 3. 创作 App

### 3.1 日记

按视角角色分册，AI 生成私密日记。

**页面流**：
```text
角色书架 → 日记目录 → 日记详情
                   ↘ 生成 / 批量生成 / 让他人阅读
```

**功能**：
- 按视角角色自动分册
- 目录：标题搜索、正序/倒序
- 详情：标题、时间、来源、正文；上/下条、置顶/底、复制、收藏、编辑、删
- 可删除整本日记
- **生成**：选来源楼层 → AI 写日记 → 预览/直接保存
- **批量生成**：楼层表达式 + AI/User 楼层筛选 + 逐层或分组 + 进度
- **让他人阅读**：选多篇日记 + 阅读者 → 生成带标识的反馈日记

### 3.2 番外

按类型分册，章节式创作管理。

**页面流**：
```text
番外书架 → 章节目录 → 章节详情
                   ↘ 新建 / 续写 / 重写 / 批量 / 章节总结
```

**功能**：
- 按番外类型分册（默认类型 + 自定义类型）
- 类型提示词支持 `{{char}}`、`{{user}}` 替换
- 目录：搜索、正序/倒序
- 番外大纲
- AI 章节总结（保存于书级，可启用/查看/删）
- 续写时可用启用的总结替代已覆盖的旧章节（未覆盖的章节仍发原文）
- 详情：字号/行距、上/下章、置顶/底、复制、收藏、编辑、删
- 批量生成：全部/单层/范围 + 按范围分组或指定章数

**续写提示词顺序**：
```
前文内容（续写点之前的全部章节或总结替代部分）
  → 引用素材（如有）
  → App 提示词
  → 番外类型提示词
  → 用户额外要求
  → 固定输出格式
```

### 3.3 论坛

生成论坛帖子和楼层回复。

**页面流**：
```text
板块总览 → 帖子列表 → 帖子详情 → 生成回复 / 手动回复
```

**功能**：
- AI 生成板块名、帖子标题、作者、主楼和首批回复
- 排序：最新回复、最新发布、热度、收藏
- 帖子详情展示主楼 + 普通回复 + 楼中楼关系
- 复制主楼或单条回复
- 收藏、编辑、删帖
- 继续生成回复 + 手动添加回复 + 回复指定楼层

### 3.4 小剧场

生成可渲染的短篇内容，支持 Markdown 和前端 HTML 两种模式。

**功能**：
- 按类型管理，按使用频率排序
- 自定义类型
- 配置：参与角色、类型提示词、渲染模式
- **Markdown 模式**：`<content>` 内为文本/Markdown，用手机自带阅读器渲染
- **前端渲染模式**：`<content>` 内为 HTML，在沙盒 iframe 中独立渲染
  - `sandbox="allow-scripts"`（不含 `allow-same-origin`）
  - CSP：`default-src 'none'; script-src 'unsafe-inline' https:; style-src 'unsafe-inline' https:; img-src https: data: blob:; media-src https: data: blob:; font-src https: data:; connect-src 'none'; frame-src 'none'; object-src 'none'; base-uri 'none'; form-action 'none'`
  - 父子页面仅通过 `postMessage` 传递内容高度，不开放 DOM 访问
  - 切换详情/关闭手机时卸载 iframe，不持久化 iframe DOM 或运行状态
- 详情：复制、收藏、编辑、删
- 生成历史可搜索

### 3.5 书信

按角色对组合整理信件。

**功能**：
- 按角色对分册（参与者组合排序生成稳定 key，发信方向不影响归属）
- 格式：正式信 / 便条 / 短信 / 邮件
- 目录正序/倒序
- 详情：上/下封、置顶/底、复制、收藏、编辑、删
- 生成回信：自动携带最近 N 封相关往返信件作为上下文
- 可删除整本书信

### 3.6 总结

按楼层范围生成剧情总结。

**功能**：
- 按总结集管理条目
- 单次范围总结 + 分批总结（带进度）
- 详情：复制、收藏、编辑、删
- 不提供导入世界书

## 4. 工具 App

### 4.1 收藏

汇总跨聊天收藏。

**功能**：
- 聚合日记/番外/论坛/小剧场/书信/总结的已收藏内容
- 按内容类型、来源角色、聊天筛选
- 当前聊天中的收藏可跳转原详情页
- 其他聊天的收藏打开只读快照
- 快照跟随原文：原文编辑时同步更新；取消收藏/删原文时同步删快照
- 支持批量取消收藏

### 4.2 提示词工坊

管理创作提示词和快速短语。

**功能**：
- App 提示词（日记/论坛/番外/小剧场/书信/总结 + 日记阅读反应 + 番外章节总结）
- 类型提示词（番外/小剧场），可新增/改/删/预览
- 快速短语分组管理，可一键追加到「额外要求」
- 选择性导出/导入

### 4.3 聊天统计

当前聊天的数据分析。

**功能**：
- 总楼层、用户楼层、AI 楼层、总字数
- 每楼字数分布
- 角色发言统计
- 切聊天后自动刷新

### 4.4 阅读聊天

读取当前角色卡全部聊天的 AI 输出。

**功能**：
- 列出当前角色卡的聊天文件目录（文件名、更新时间、总楼层数）
- 进入某聊天后只展示 AI 楼层（过滤用户消息、系统消息、隐藏楼层）
- 每楼展示为标题 + 正文
- 用户可设置标题正则和正文正则（各一组，含 find/replace/flags）
- 正则替换在独立 Web Worker 中执行，`1500ms` 超时终止，防止灾难性回溯
- 虚拟滚动渲染长列表
- 切换角色卡后清空缓存

> 阅读聊天和番外的目录-详情结构一致，正则替换只是方便用户确认每楼标题和正文的展示形式。

### 4.5 八股去除

内置于生成流程：每次生成完成后，自动对生成正文执行八股检测，展示命中提示。App 本身只负责规则管理。

**生成后检测流程**：
```text
生成完成 → 解析得到正文
  → 用启用的规则扫描正文
  → 命中时展示：命中位置、匹配原文、建议替换、上下文高亮
  → 用户选择：应用替换 / 不应用 / 编辑后再应用
```

- 不静默改写
- 预览页和详情页均可触发检测

**规则管理（八股 App）**：
- 词汇替换规则 + 句式模板规则
- 规则可启用/停用/新增/编辑/删/恢复默认
- 规则持久化

## 5. 设置 App

全部设置在手机内完成，不在酒馆扩展设置页创建面板。

- **数据统计**：各类内容数量、当前聊天数据大小
- **数据操作**：完整 ZIP 导出/导入、清空当前聊天创作数据
- **生成默认值**：来源楼层模式、额外上下文、结果去向（预览/保存）、流式开关
- **外观**：亮/暗主题、壁纸（预设 + 自定义上传）
- **阅读器**：字号、行高
- **文本通道**：酒馆当前 API / 外部 OpenAI 兼容 API
- **悬浮球**：开关、尺寸、颜色
- **主屏布局**：App 显隐、排序、Dock、布局重置
- **数据恢复**：异常恢复日志管理
- 破坏性操作须二次确认

## 6. 通用生成能力

所有创作 App 共用一套生成管线，不各自实现。

### 6.1 生成流程

```text
选择来源楼层
  → 可选引用手机内已有创作
  → 快照全部楼层的原始隐藏状态并写入恢复日志
  → 临时隐藏来源范围以外的可见楼层
  → 按顺序拼接 user_input
  → 调用 TavernHelper generate({ user_input, should_silence: true, should_stream, generation_id })
  → 当前酒馆自动应用角色卡/预设/世界书
  → 无论成功、停止或失败，都恢复全部楼层的原始隐藏状态
  → 解析返回的 XML
  → 预览 / 直接保存
```


### 6.2 user_input 拼接顺序

来源楼层由酒馆原生聊天历史机制处理（临时隐藏事务确保只有来源楼层可见），**不拼入 `user_input`**。

`user_input` 的拼接模板：

```
额外上下文（如有）
  → 引用素材（如有）
  → App 提示词
  → 类型提示词（如有）
  → 用户额外要求
  → 固定输出格式（XML 模板，由解析器维护，不可用户编辑）
```

空的可选区段直接省略。

**各动作的额外上下文**（位于引用素材之前、App 提示词之前）：

| App | 动作 | 额外上下文 |
|-----|------|-----------|
| 日记 | 让他人阅读 | 被阅读的日记正文 |
| 论坛 | 续回 | 主楼 + 现有回复（含楼中楼关系） |
| 番外 | 续写 | 续写点前全部章节（书级总结可替代已覆盖部分） |
| 番外 | 重写 | 原章节正文 |
| 书信 | 回信 | 最近 N 封相关往返信件 |
| 其他 | 普通生成 | 无 |

番外和小剧场使用类型提示词，其余 App 不使用。

### 6.3 来源楼层选择

来源楼层表示「本次创作基于哪些现有聊天楼层」。支持六种模式：

- 最新楼层
- 从 0 至指定楼层
- 全部楼层
- 指定单层
- 最近 N 楼
- 自定义范围

**读取规则**：

- 只读可见楼层、只取当前选中 swipe，不发送隐藏楼层或未选中 swipe
- 范围解析后无任何可见楼层 → 阻止生成并提示
- 不重新组装 `{role, content}`，不改写正文，不附加楼层号或发言者标签
- 由酒馆原生流程保留群聊角色、当前 swipe、正则和世界书处理

### 6.4 来源楼层隐藏事务

为了让生成时的聊天历史只包含用户选中的来源楼层，需要在调用 `generate()` 之前临时隐藏来源范围外的楼层，生成结束后恢复原状。

**事务流程**：

```text
快照全部楼层的 message_id + is_hidden 状态
  → 将快照（含 contentHash）持久化为恢复日志
  → 只把「原本可见且不在来源集合内」的楼层临时设为隐藏
  → 原本隐藏的楼层始终保持隐藏
  → 若来源集合已覆盖全部原本可见楼层，则跳过隐藏
  → 调用 generate()
  → finally：按快照逐层恢复全部楼层的原始隐藏状态
  → 核验恢复后的身份哈希与快照匹配
  → saveChat() 将恢复后的状态落盘
  → 清除恢复日志
```

**关键约束**：

- 隐藏/恢复统一使用 `setChatMessages(..., { refresh: 'none' })` 避免页面闪烁
- 临时隐藏阶段**禁止**调用 `saveChat()`；只有 `finally` 中恢复并核验后才保存
- 恢复必须写在 `finally` 中：成功、停止、失败、解析异常都执行
- 恢复时按快照逐层还原，不能简单把所有楼层取消隐藏
- 来源范围包含全部可见楼层时，不创建恢复日志、不执行隐藏

**scope 安全**：

- 事务启动时绑定当前 `scopeId` 和任务 ID
- 恢复前再次核对 `chat_metadata` 中的 `scopeId`，禁止跨聊天恢复
- 生成期间发生聊天切换 → 立即停止生成，保留恢复日志，新聊天激活时优先恢复

**崩溃恢复日志**（按 `scopeId` 保存在 `extension_settings.sillytavernPhone.pendingVisibilityRecoveries`）：

```ts
interface PendingVisibilityRecovery {
  scopeId: string;
  generationId: string;
  createdAt: string;
  messages: Array<{
    messageId: number;
    isHidden: boolean;
    role: 'system' | 'assistant' | 'user';
    name: string;
    contentHash: string;  // SHA-256 of role + name + 当前swipe正文
  }>;
}
```

- 必须先持久化恢复日志成功，才能隐藏楼层
- `contentHash` 使用 Web Crypto SHA-256
- 恢复日志若超过 settings 可保存上限，则阻止本次生成
- 恢复后身份哈希不匹配 → 保留日志进入人工恢复，不盲目写回楼层号
- 插件启动和聊天切换后先检查 `pendingVisibilityRecoveries[currentScopeId]`，自动核对恢复
- 日志超过合理时限仍需用户确认放弃，不能自动丢弃

### 6.5 并发控制与 SendGuard

- 全插件只有一把来源隐藏互斥锁，同一时间只允许一个生成事务
- 启动前若本插件或外部已有生成任务 → 拒绝开始
- 生成期间由 `SendGuard` 拦截酒馆 UI 发送按钮和聊天输入框 Enter 发送
- `SendGuard` 添加带插件命名空间的捕获阶段监听和提示遮罩
- **不调用**全局 `deactivateSendButtons()`/`activateSendButtons()`，避免影响其他扩展或酒馆自身
- 检测到非本任务的生成 → 立即停止手机任务并先恢复楼层
- 事务结束只移除自己的监听和遮罩，不负责「重新启用」酒馆按钮

### 6.6 生成参数

```ts
interface GenerationConfig {
  sourceMode: 'latest' | 'fromStart' | 'all' | 'single' | 'recent' | 'range';
  sourceRanges: Array<{ start: number; end: number }>;
  resultMode: 'preview' | 'save';   // 预览后确认 / 解析后直接保存
  stream: boolean;                   // 是否流式传输
}
```

- `preview`：解析后进预览页，确认后保存
- `save`：解析成功后直接保存并跳转详情
- `stream`：显示实时文本，结束后按 resultMode 处理
- 用户停止后不按 `save` 自动保存；有流式文本时作为未完成草稿进入修复，无文本时仅记录终止状态

### 6.7 引用素材

- 可引用手机内已有的日记/论坛/番外/小剧场/书信/总结
- 引用选择器按内容类型和分册分组
- 支持搜索、展开/收起、全选
- 引用放入 `user_input` 参考资料区，不伪装成聊天楼层
- 各动作的固定附带内容（均为手机内部已保存数据，非聊天楼层）：
  - **番外续写**：续写点前全部章节（书级总结可替代已覆盖部分）
  - **论坛续回**：主楼 + 现有回复（含楼中楼关系）
  - **书信回信**：最近 N 封相关往返信件
  - **日记阅读反应**：被阅读的这一篇日记

### 6.8 解析

所有创作使用统一的 XML 输出格式：

```xml
<!-- 日记 / 番外章节 / 小剧场 / 总结 -->
<result>
  <title>标题</title>
  <content>正文（小剧场 frontend 模式时为 HTML）</content>
</result>

<!-- 论坛首帖 -->
<result>
  <board>板块名</board>
  <title>帖子标题</title>
  <author>作者</author>
  <content>主楼正文</content>
  <replies>
    <reply><author>作者</author><content>回复</content><parent_ref></parent_ref></reply>
  </replies>
</result>

<!-- 书信 -->
<result>
  <title>标题</title>
  <content>正文</content>
</result>
```

- 解析器提取第一个完整 `<result>`，忽略外围说明文字
- 多个 `<result>` 或缺少必需字段 → 进入解析修复
- 小剧场 frontend 模式以 `<content>` 起止标签边界提取原始 HTML
- 解析失败不丢原文，作为失败草稿可编辑后重新解析
- 论坛的临时引用（`E1/E2/N1/N2`）在保存时转为真实 `parentReplyId`；自引用/循环/不存在降级为普通回复

### 6.9 文本生成通道

```ts
type TextProvider =
  | { mode: 'tavern' }
  | { mode: 'external'; apiUrl: string; apiKey?: string; model: string; contextWindow?: number; maxOutputTokens?: number };
```

- **tavern**：`generate()` 不传 `custom_api`，使用酒馆当前 API/模型/采样参数
- **external**：`generate({ custom_api: { apiurl, key, model, source: 'openai', max_tokens? } })`，其余管线不变
- URL 规范化：去首尾空白、末尾 `/`、`/models`、`/chat/completions`
- URL 只允许 `http:`/`https:`，拒绝内嵌用户名密码和危险协议
- API Key 遮罩显示，不写日志，默认导出脱敏
- 连接测试和模型刷新通过 TavernHelper `getModelList({ apiurl, key })` 代理
- 外部 API 未配置或测试失败时阻止该通道生成

## 7. 数据持久化

### 7.1 存储方式

数据优先保存为 SillyTavern `user/files` 下的独立 JSON 文件，`extension_settings.sillytavernPhone` 作为读写失败的兜底。这与 Diary 参考项目的存储机制一致：

- `/api/files/upload` — POST 上传文件（需 `getRequestHeaders()` 认证）
- `/api/files/verify` — POST 检查文件是否存在
- `/api/files/delete` — POST 删除文件
- `/{path}` — GET 读取文件（`cache: 'no-store'`）

所有 revision 数据文件使用不可变命名（含 `revisionId`），保存时先写新文件并回读校验，再一次性更新 storage index 指针。

### 7.2 数据分布

| 数据 | 存储位置 | 作用域 |
|------|---------|--------|
| 活动文件指针和 scope 绑定 | `user/files/phone-storage-index.json` | 全局 |
| 日记/论坛/番外/小剧场/书信/总结 | `user/files/phone-{scopeId}-{type}-{revisionId}.json` | 单聊天 |
| 收藏快照/偏好/外部 API/提示词/八股规则/壁纸索引 | `user/files/phone-global-{domain}-{revisionId}.json` | 全局 |
| 自定义壁纸原文件 | `user/files/phone-wallpaper-{id}.{ext}` | 全局 |
| 恢复日志/操作日志/兜底数据 | `extension_settings.sillytavernPhone` | 全局 |

### 7.3 聊天作用域

通过稳定 `scopeId` 定位聊天级数据：

1. 首次进入聊天时，通过 `getContext()` 取得 `chat_metadata`
2. 写入 `chat_metadata.sillytavernPhone.scopeId`（新 UUID）
3. 使用 `saveMetadata()` 持久化到 metadata
4. 在 `phone-storage-index.json` 中记录 `scopeBindings[scopeId]`

- `ownerKey`：单角色聊天用 `char:{avatarId}`，群聊用 `group:{groupId}`，不依赖易变的 `characterId`
- 聊天改名沿用原 `scopeId`，不重新建库
- Branch/Checkpoint 创建新 `scopeId` 并可选复制原数据

### 7.4 数据原则

- 所有持久数据带 `schemaVersion`，通过显式迁移升级
- ID 使用 UUID，时间使用 ISO 8601
- 删除分册时同步清理关联的收藏快照
- 导入先完整校验和迁移再一次性提交
- 自定义壁纸不塞进 JSON base64，保存为独立原文件
- 完整备份使用 ZIP 包（含 manifest.json + 数据 JSON + 壁纸原文件）

### 8.3 分层职责

- **组件**：展示 + 触发动作，不直接读写 SillyTavern API
- **Store**：UI 状态 + 业务编排
- **Service**：无界面能力（楼层、生成、解析、剪贴板）
- **Repository**：持久化、迁移、导入导出、作用域切换
- App 间只通过公共服务、实体 ID 和导航参数协作，不互相导入内部实现

### 8.4 App 扩展契约

```ts
interface PhoneAppDefinition {
  id: string;
  name: string;
  icon: string;
  defaultRoute: string;
  defaultVisible: boolean;
  defaultDock: boolean;
  defaultOrder: number;
  load: () => Promise<PhoneAppModule>;
}

interface PhoneAppModule {
  routes: AppRouteDefinition[];
  generationAdapters?: GenerationAdapter[];
  storageDomains?: StorageDomainDefinition[];
  dispose?: () => void;
}
```

- 内置 App 使用稳定 ID，注册时校验唯一性
- 新 App 只需注册定义 + 路由 + 可选的 storage domain/generation adapter
- 新增 App 无需修改手机壳、导航、生成内核或 storage index

### 8.5 生命周期

1. `APP_READY` → 等待 `window.TavernHelper` → 初始化
2. 注册扩展菜单入口 + 可选悬浮球
3. 首次打开时创建 Vue 根节点，后续只显隐不重建
4. 关闭：保留导航栈和表单状态，卸载 iframe
5. 销毁：彻底卸载 Vue + 清理节点 + 解除监听
6. 监听聊天切换 → 刷新聊天级数据
7. 提供显式 `destroy()` 用于热重载/禁用

- lint 0 error / 0 warning
